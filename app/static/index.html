일괄 소스 보강?

간편 등록 후 미연결 상품을 모아 Discogs / ManiaDB / Aladin 후보로 교체합니다. 바코드 일치 또는 후보 1건은 자동 선택됩니다.
소스 보강 활용 매뉴얼
  1. 메타가 부족하거나 외부 소스를 다시 연결해야 할 때 이 화면을 엽니다.
  2. 보강 대상(`미연결만`, `전체`, `기연결만`)과 개수를 먼저 정합니다.
  3. `대상 불러오기` 후 `전체 후보 조회`로 후보를 채우고, 자동 선택된 후보를 확인하거나 필요한 경우 직접 고릅니다.
  4. `선택 후보 일괄 교체`로 메타를 반영한 뒤, 예외가 남으면 `검색/관리` 또는 `예외 큐`로 이어갑니다.
예외 큐의 `소스 미연결`, `커버 없음`, `곡 정보 없음`과 가장 자주 연결되는 작업대입니다.

  1. 새 장식장을 만들거나 장식장 정책을 바꿔야 할 때 이 화면을 엽니다.
  2. 이름, 규격, 층 수, 칸 수와 함께 정렬 정책을 먼저 정합니다.
  3. `장식장 생성`으로 슬롯 구조를 일괄 만들고, 필요하면 아래 목록에서 구조를 다시 확인합니다.
  4. 장 교체나 증설은 구조를 먼저 만든 뒤 `대시보드`에서 실제 이동을 진행합니다.
전시형 장식장은 `아티스트/발매/앨범`, 증설·보관 장식장은 `등록 코드 순`이 운영에 유리합니다.

카메라 페이지 활용 매뉴얼
  1. 장식장 확인용 카메라를 장식장 단위로 연결할 때 이 화면을 엽니다.
  2. 장치 검색 또는 ONVIF 장치 URL 입력 후 `연결 테스트/자동 채움`으로 스냅샷/스트림 주소를 먼저 확인합니다.
  3. 장식장명과 카메라명을 확인한 뒤 저장하면 `대시보드`에서 같은 장식장의 스냅샷을 바로 볼 수 있습니다.
  4. 촬영 각도 점검은 실제 장식장을 선택한 뒤 `스냅샷 새로고침` 또는 `영상 열기`로 확인합니다.
이 화면은 NVR 관리가 아니라 장식장 확인 보조용입니다. 스냅샷 확인이 우선이고, 녹화/이벤트 기능은 다루지 않습니다.

장식장 확인 카메라?

장식장마다 확인용 카메라 1개를 연결합니다. 대시보드에서는 연결된 장식장의 스냅샷만 표시합니다.
같은 네트워크 대역의 ONVIF 카메라를 찾습니다. 검색 결과에서 `등록폼 채우기`를 누르면 장치 URL과 카메라명이 입력됩니다.

슬롯 페이지 활용 매뉴얼
  1. 비정형 구조나 예외 칸을 직접 수정해야 할 때 이 화면을 엽니다.
  2. 일괄 생성으로 해결 가능한 구조는 먼저 `장식장` 화면에서 처리합니다.

  4. 수정 후에는 `대시보드`에서 실제 배치 화면에 반영됐는지 바로 확인합니다.
전체 구조를 이 화면에서 처음부터 만들기보다, 예외 칸만 보정하는 용도로 쓰는 편이 훨씬 단순합니다.

예외 큐 페이지 활용 매뉴얼
  1. 운영 병목이 되는 상품만 따로 모아 정리할 때 이 화면을 엽니다.
  2. 예외 종류를 먼저 선택하고, 자주 쓰는 조합은 저장 필터로 남깁니다.
  3. 다중 선택 후 `소스 보강`, `실물 기준 맞춤`, `마스터 정리` 같은 다음 작업대로 바로 넘깁니다.
  4. 정리 후에는 관련 탭에서 결과를 처리하고 다시 돌아와 남은 예외를 줄여갑니다.
예외 큐는 병목 제거용 허브입니다. 정상 상품까지 여기서 다루기보다, 문제 상품만 빠르게 쳐내는 용도로 쓰는 편이 좋습니다.

계정 페이지 활용 매뉴얼
  1. 관리자나 현장 운영자 계정을 추가하거나 점검할 때 이 화면을 엽니다.
  2. 기본 시스템 계정은 읽기 전용으로 두고, 새 계정만 여기서 관리합니다.

권한은 필요한 만큼만 주는 편이 안전합니다. 특히 현장 운영 계정은 조회/로그 중심으로 두는 것이 좋습니다.

백업/내보내기 페이지 활용 매뉴얼
  1. 대량 수정이나 장식장 재배치처럼 되돌리기 어려운 작업 전에는 이 화면을 먼저 엽니다.
  2. DB 백업을 먼저 받고, 필요한 경우 운영 공유용 목록은 CSV로 내보냅니다.
  3. 문제가 생기면 백업 파일과 launchd 로그를 함께 확인해 복구 경로를 잡습니다.
  4. 백업을 받은 뒤에만 다른 운영 탭으로 넘어가 실제 작업을 시작합니다.
백업은 보험에 가깝습니다. 바쁘더라도 장식장 구조 변경이나 대량 수정 전에는 먼저 받아두는 편이 안전합니다.

백업 / 내보내기?

대량 작업 전에는 DB 백업을 먼저 받고, 운영 공유용 목록은 CSV로 내보냅니다.
보유 상품 CSV에는 위치, 상태, 구매처, 권장 보관 규격, 기본 메타가 포함됩니다. 마스터 CSV에는 마스터 ID, 소스, 멤버 수가 포함됩니다.
메타 동기화 페이지 활용 매뉴얼
  1. 연결된 외부 소스 상품의 누락 메타를 묶어서 보강해야 할 때 이 화면을 엽니다.
  2. 기본은 `누락 필드만 보강`으로 두고, 정말 필요할 때만 전체 동기화를 실행합니다.
  3. 커버, 바코드, 카탈로그, 트랙리스트가 비어 있는 상품이 쌓였을 때 우선 사용합니다.
  4. 실행 후에는 상태 요약과 `검색/관리` 화면에서 실제 반영 결과를 확인합니다.
전체 동기화는 범위가 큽니다. 평소에는 누락 필드만 보강하고, 대규모 재정리는 별도 점검 후 실행하는 편이 안전합니다.
