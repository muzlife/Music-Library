# 미디어 > 관리 패널 재설계

## 목표

현재 2단 레이아웃(왼쪽 마스터 정보 / 오른쪽 상품 수정)을 단일 컬럼 섹션 스택으로 전환한다.
주 작업(상품 수정)이 최상단에 전체 너비로 펼쳐지고, 빈도 낮은 기능은 접힌 상태로 하단 배치.
불필요한 인라인 설명은 제거하거나 풍선 도움말(help-dot)로 이동한다.

---

## 현재 구조의 문제

| 문제 | 내용 |
|------|------|
| 2단 레이아웃 | 왼쪽(마스터) / 오른쪽(상품) 분리로 화면 너비 낭비, 스크롤 불일치 |
| 불필요한 접기 단계 | `수정/편집` 상위 `<details>` → 그 안에 다시 `<details>` (2단계 접기) |
| 리뷰·Spotify 접근성 저하 | 마스터 섹션 기본 닫힘이면 자주 쓰는 리뷰·Spotify가 묻힘 |
| 섹션 번호 | `2-1`, `2-2`, `2-3` 헤딩이 관료적 인상 |
| 중복 인라인 텍스트 | help-dot와 동일 내용을 `.mini` 텍스트로 반복 |

---

## 새 레이아웃 — 6개 섹션 단일 컬럼

```
[선택된 상품 레이블]                   ← 항상 표시

▼ 상품 수정          [저장]           ← 기본 열림
▼ Spotify · 리뷰                      ← 기본 열림
▶ 마스터 메타 수정                    ← 기본 닫힘
▶ 위치                                ← 기본 닫힘
▶ 보유 상품                           ← 기본 닫힘
▶ 추가 · 연계                         ← 기본 닫힘
```

`<details>` 요소를 각 섹션의 루트로 사용. `open` 어트리뷰트로 기본 상태 제어.

---

## 섹션별 내부 구성

### ① 상품 수정 (기본 열림)

**포함 내용 (현재 `homeEditorRightPane` 전체):**

```
소스 메타 요약 카드
  - 아티스트 · 제목 · 발매 · 레이블 (읽기 전용 요약)
  - [▶ 트랙 목록] (내부 접기 유지)
  - 힌트 "외부 소스 메타는 후보 교체로 갱신" → help-dot 이동

상품 정보 필드
  - 아티스트명 / 상품명 / 도메인코드 / 발매일
  - 레이블 / 카탈로그 / 바코드 / 스타일
  - [새상품] [홍보용] [한정판] 체크박스

미디어 · 패키징
  - 미디어 유형 / 디스크 수 / 런아웃·프레싱 (바이닐)
  - 패키징 형태 / 구성품 체크박스 목록

커버 이미지
  - 로컬 이미지 / 붙여넣기 / URL 다운로드

[▶ 구매 · 컨디션 · 메모]               ← <details> 내부 접기 유지

[저장]                                  ← 섹션 <summary> 우측에 고정 배치
```

저장 버튼을 `<summary>` 우측에 두어 섹션이 열려 있는 동안 항상 접근 가능.

---

### ② Spotify · 리뷰 (기본 열림)

**마스터 컨텍스트 (읽기 전용, compact):**
- 커버 썸네일 + 아티스트 — 앨범명 + 발매연도
- 현재 `homeSection1MasterInfo`의 카드 정보를 compact 형태로 재사용
- Discogs / ManiaDB 외부 링크 아이콘 포함

**Spotify 연결:**
- Album ID 입력 필드 + [연결] [해제] 버튼
- 연결 상태 표시

**앨범 리뷰:**
- 현재 `homeMasterReviewSection` 구현 그대로
- 리뷰 없음: [Wikipedia 자동수집] [URL 입력] [직접 입력]
- 리뷰 있음: 미리보기 + 출처 + [삭제] + 교체 버튼

---

### ③ 마스터 메타 수정 (기본 닫힘)

**메타 교정 필드:**
- 아티스트 재정의 / 앨범명 재정의 / 발매연도 / 도메인 / 교정 메모
- 각 필드에 help-dot 유지 (현재 구현 그대로)
- 소스 기준 힌트 텍스트 (`homeMasterCorrectionSourceHint`) 유지

**정렬 아티스트명:**
- 입력 필드 + [저장] 버튼
- 중복 note 텍스트(`같은 마스터의 모든 미디어 정렬에 공통 적용됩니다`) **제거** (help-dot에 이미 있음)

**[저장] 버튼**

**마스터 삭제 (섹션 하단, 구분선 후):**
- [연결 상품 함께 삭제 □] 체크박스 + [삭제] 버튼
- 인라인 경고 문구(`상품 편집을 먼저 마친 뒤, 정말 필요할 때만 사용하세요...`) → help-dot 이동

---

### ④ 위치 (기본 닫힘)

현재 `homeCabinetSection` 내용 그대로:
- 장식장 미니맵 / 선반 트랙 시각화
- [이전 위치 복구] [대시보드에서 이동] 버튼

### ⑥ 추가 · 연계 (기본 닫힘)

```
[▶ 마스터 연계 추가]                    ← 내부 접기
  설명 단락 → help-dot 이동

[▶ 다른 소스에서 추가]                  ← 내부 접기
  바코드 조회 / 무바코드 조회
  설명 단락 → help-dot 이동

[▶ 수집품 연계]                         ← 내부 접기
  안내 단락 2개 → help-dot 이동
```

## 텍스트 정리 방침

### 완전 제거

| 위치 | 제거 대상 | 이유 |
|------|-----------|------|
| Sort artist 행 | `.dashboard-selected-sort-artist-note` | help-dot에 동일 내용 |
| 섹션 헤딩 | `2-1`, `2-2`, `2-3` 번호 | 단순 제목으로 대체 |
| `homeEditUnifiedDetails` | `수정 / 편집` summary 레이블 | 섹션 구조 해체로 불필요 |

### help-dot(?) 버튼으로 이동

| 위치 | 이동 대상 텍스트 |
|------|-----------------|
| 소스 메타 요약 | `외부 소스 메타는 후보 교체로 갱신` |
| 마스터 연계 추가 | `필요할 때만 펼쳐 현재 상품과 연결할 마스터 후보를 확인합니다...` |
| 다른 소스에서 추가 | `현재 마스터에 없는 상품을 다른 소스에서 찾아 연계 상품으로 추가합니다...` |
| 상품 관계 | `마스터, 시리즈, 박스세트, 연관 발매를 함께 관리` |
| 수집품 연계 | 안내 단락 2개 전체 |
| 마스터 삭제 | `상품 편집을 먼저 마친 뒤, 정말 필요할 때만 사용하세요...` |

### 유지

- 각 입력 필드의 help-dot (현재 구현 그대로)
- 에러 / 상태 메시지
- 소스 기준 힌트 (`homeMasterCorrectionSourceHint`) — 동적 정보라 유지

## 구현 접근 방식

**단계적 교체** — 기존 ID·클래스를 최대한 유지하고 감싸는 구조(`<details>`, `<div>`)만 재배치.
JS 이벤트 핸들러는 element ID 기준으로 동작하므로 ID 유지 시 JS 수정 최소화.

예외: `homeEditorLeftPane` / `homeEditorRightPane` div는 제거. 내부 콘텐츠는 새 섹션 `<details>`로 이동.

---

## 성공 기준

- 상품 수정 섹션이 선택 시 자동으로 열려 있고 전체 너비 사용
- Spotify 연결과 리뷰 관리가 스크롤 없이 기본 접근 가능
- 마스터 override 필드는 닫힌 상태로 시작
- 인라인 설명 텍스트 6개 이상 제거 또는 help-dot 이동
- 기존 JS 이벤트 핸들러 동작 유지 (ID 보존)
