# 운영 홈 헤더 디자인 1차 설계

## 목적
- `운영 홈(/ops)`에만 적용되는 별도의 메인 헤더를 추가한다.
- 기존 `관리 콘솔` 헤더와 시각적으로 분리해, 현장 운영 화면이 더 친근하고 덜 딱딱하게 느껴지도록 만든다.
- 검색 중심 동선을 해치지 않으면서도 운영 홈 진입 시 분위기 전환이 즉시 느껴지게 한다.

## 배경
현재 운영 홈은 관리 화면에서 사용하던 구조를 일부 그대로 재활용하고 있다. 기능은 분리되고 있지만 시각적으로는 여전히 관리 도구의 연장처럼 보인다. 사용자는 운영 홈에서 검색과 위치 확인에 집중해야 하므로, 운영 홈은 더 부드럽고 친근한 인상을 주는 것이 적합하다.

## 확정된 방향
- 분위기: `레코드 재킷 질감 + 작은 음표 포인트`
- 제목: `라이브러리 운영 홈`
- 적용 범위: `운영 홈(/ops)`만
- 관리 화면(`/admin`)의 기존 헤더는 유지

## 목표 경험
- 운영 홈에 들어오면 관리자용 백오피스와 다른 화면이라는 인상이 바로 전달된다.
- 검색창이 여전히 가장 중요한 첫 액션으로 보인다.
- 장식은 분위기를 만드는 용도에 그치고, 정보 밀도나 조작 흐름을 방해하지 않는다.

## 디자인 원칙
1. 운영 홈 전용
- 새 헤더는 `운영 홈(/ops)`에서만 보인다.
- `장식장(/ops/cabinets)`에서는 같은 계열 톤을 유지하되, 더 얕은 헤더 또는 간략한 상태 바로 확장할 수 있다. 1차에서는 운영 홈에만 적용한다.

2. 관리 화면과 명확히 분리
- `관리 화면(/admin)`은 기존의 도구형 헤더를 유지한다.
- 운영 홈은 더 따뜻하고 부드러운 색과 질감을 사용한다.

3. 검색 우선 유지
- 새 헤더는 검색창을 아래로 과하게 밀지 않는다.
- 헤더 높이는 얕고 넓게 유지한다.
- 운영 홈 첫 스크롤 안에서 제목과 검색창이 함께 보여야 한다.

4. 일러스트는 얇게
- 사진이나 복잡한 캐릭터형 그래픽은 사용하지 않는다.
- 작은 음표, 점선, 종이/재킷 질감, 얕은 색면 정도로 제한한다.
- 시선은 제목과 검색창으로 돌아오게 해야 한다.

## 레이아웃
### 1. 헤더 배너
운영 홈 상단에 가로형 배너를 배치한다.

구성:
- 좌측
  - 제목: `라이브러리 운영 홈`
  - 짧은 설명 문구
    - 예: `상품을 찾고, 위치를 확인하고, 장식장으로 바로 이동합니다.`
- 우측
  - 작은 장식 일러스트
  - 레코드 재킷 종이 질감 블록
  - 작은 음표와 선율 점 패턴

### 2. 유틸리티 영역
기존 상단 유틸 바는 유지하되, 운영 홈 헤더와 충돌하지 않게 얇게 배치한다.

포함 요소:
- 세션 정보
- 로그아웃
- 관리자만 `관리`

배치 원칙:
- 유틸 바는 도구성 정보
- 새 헤더는 분위기와 맥락 정보
- 둘은 역할이 다르므로 한 블록으로 합치지 않는다.

### 3. 검색과의 연결
- 새 헤더 바로 아래에 검색 카드가 이어진다.
- 시각적으로는 헤더와 검색 카드가 한 세트처럼 보이되, 검색 카드가 더 높은 대비를 가진다.
- 사용자가 헤더를 읽지 않아도 곧바로 검색을 시작할 수 있어야 한다.

## 시각 방향
### 색감
- 기본 톤: 따뜻한 베이지, 소프트 민트, 옅은 블루
- 포인트 톤: 어두운 청록 또는 남청
- 관리 화면처럼 강한 teal hero를 재사용하지 않는다.

### 텍스처
- 종이/재킷 같은 미세 질감 배경
- 너무 현실적인 사진 텍스처가 아니라 CSS로 표현 가능한 얕은 패턴 중심

### 장식 요소
- 작은 음표 아이콘
- 얇은 선율 점/별점 형태 패턴
- 재킷 모서리를 연상시키는 라운드 사각형 레이어

### 타이포그래피
- 현재 프로젝트의 텍스트 체계를 유지한다.
- 별도 폰트 도입은 하지 않는다.
- 제목만 조금 더 존재감 있게 조정한다.

## 반응형 원칙
### 데스크톱
- 좌측 텍스트 / 우측 장식
- 검색 카드가 바로 아래에 붙는다.

### 태블릿/모바일
- 우측 장식 면적 축소
- 패턴 수를 줄여 시야 방해를 줄인다.
- 제목과 설명을 우선 보이게 하고, 검색창이 첫 스크롤 안에 유지되게 한다.

## 구현 범위
### 포함
- 운영 홈 전용 헤더 마크업 추가
- 관련 CSS 추가
- 운영 홈에서만 보이도록 route/shell mode 조건 연결
- 관리 화면 hero와 운영 홈 hero의 공존 정리

### 제외
- 장식장 화면 별도 헤더 재설계
- 관리자 화면 톤 변경
- 새로운 이미지 자산 다운로드
- 외부 SVG/폰트 의존성 추가

## 구현 접근안 비교
### 안 1. 배너형 일러스트 헤더
- 운영 홈 상단에 얕은 가로 배너 추가
- 장점
  - 관리 화면과 차별감이 큼
  - 검색 중심 구조를 유지하기 쉬움
  - 기존 레이아웃 변경이 비교적 작음
- 단점
  - 시각 요소를 과하면 산만해질 수 있음

### 안 2. 카드형 인사 헤더
- 검색 카드 위에 작은 환영 카드 배치
- 장점
  - 안전하고 구현이 단순함
- 단점
  - 메인 헤더로 느껴지는 힘이 약함

### 안 3. 배경 질감만 추가
- 제목/구조는 그대로 두고 배경만 변경
- 장점
  - 리스크가 가장 낮음
- 단점
  - 변화 체감이 작음

## 채택안
- `안 1. 배너형 일러스트 헤더`

채택 이유:
- 사용자 요청인 `음표들이 포함된 아기자기한 이미지` 요구를 가장 자연스럽게 만족한다.
- 관리 화면과 운영 홈의 분위기를 가장 명확하게 분리할 수 있다.
- 검색 중심 동선을 유지하면서도 첫 인상을 바꾸기에 적합하다.

## 성공 기준
- 운영 홈에서만 새 헤더가 보인다.
- 관리 화면에서는 기존 관리 콘솔 헤더만 유지된다.
- 운영 홈 진입 시 더 친근한 톤이 즉시 느껴진다.
- 검색 카드가 첫 액션으로 계속 유지된다.
- 모바일에서도 검색 영역이 첫 스크롤 안에서 보인다.

## 리스크 및 대응
### 리스크 1. 장식이 검색을 밀어냄
- 대응: 헤더 높이를 얕게 유지하고, 패턴/장식을 오른쪽에 몰아 둔다.

### 리스크 2. 운영 화면이 과하게 캐주얼해짐
- 대응: 아이콘 수를 줄이고, 컬러 대비를 절제한다.

### 리스크 3. 관리 화면과 스타일 충돌
- 대응: `body[data-shell-mode="ops"]` 범위 안에서만 스타일을 적용한다.
