# 카페 직원 운영 화면 — `/ops/cafe` 레이아웃 및 제어 스펙

본 문서는 카페에 비치된 태블릿 기기들과 연동되어 직원이 신청곡 대기열을 관리하고, 음원을 제어(Spotify, 로컬 파일, CD/LP 수동)하며, 실시간 고객 반응을 모니터링할 수 있는 `/ops/cafe` 화면의 상세 레이아웃 및 작동 규칙을 정의합니다.

## 2. 구성 요소별 세부 사양

### 2.1 헤더 (Header)
* **카페 운영 타이틀 및 정보**: 실시간 신청곡 대기열 개수 표시
* **상태 인디케이터**: WebSocket 및 Spotify Connect 세션의 실시간 연결 상태를 직관적으로 표시 (🟢 연결됨 / 🔴 연결 끊김)
* **설정 버튼**: 기기 매핑 패널을 열어 신규 태블릿 접속 시 테이블 번호를 지정할 수 있는 오버레이를 호출

### 2.2 좌측 영역: 신청곡 리스트 (Queue List)
* **실시간 신청곡 대기열 (Active Queue)**:
  * 최신 요청이 아래로 쌓이며, 직관적인 위아래 동선을 가집니다.
  * 테이블 번호와 신청 시간은 가독성을 위해 **Monospace 폰트**를 강제합니다.
  * **[재생]** 버튼: 클릭 시 활성화 상태로 전환되며 해당 오디오 제어가 중앙 패널에 마운트됩니다.
  * **[보류]** 버튼: 대기열 맨 아래로 보내거나 임시 보류 상태로 전환합니다.
  * **💿 CD/LP 및 Local File의 실물 위치 표시**: 음반의 소스가 CD/LP(수동)이거나 로컬 파일(Local)인 경우, DB의 `StorageSlot` 정보를 동적 조회하여 해당 음반이 꽂혀 있는 물리적 위치 코드(예: `A-2-4`)를 짙은 노란색 배지로 카드 내에 출력하여 직원이 바로 실물 음반을 찾거나 참고할 수 있도록 지원합니다.
* **오늘 완료 목록 (History List)**:
  * 오늘 재생이 완료되었거나 취소 처리된 이력을 표시합니다. (최근 15건 한정)
  * **[복구] 버튼**: 이력 카드의 우측에 배치되며, 클릭 시 해당 신청곡을 다시 대기(REQUESTED) 상태로 복원하고 실시간 신청 대기열의 가장 마지막 순서로 되돌려 놓습니다.

### 2.3 우측 상단: Now Playing 히어로 카드 (Hero Control Card)
* **앨범 아트워크**: 대형 이미지 뷰 적용 (가로/세로 180px 이상)
* **메타데이터 정보**: 제목, 아티스트, 재생 소스(Spotify, Local, CD/LP) 노출
* **공통 재생 컨트롤**: 
  * 재생/일시정지(`[⏸]`), 정지(`[⏹]`), 볼륨 제어 바 (`[🔊]`) 제공
  * Spotify API와 HTML5 오디오 태그를 추상화한 통합 플레이어 래퍼 구현
* **📝 가사 미리보기 (Lyrics Panel)**:
  * `/cafe/lyrics` API와 연계하여 현재 곡의 가사를 실시간 조회하여 콤팩트하게 표기 (직원의 선곡 응대용)

### 2.4 우측 하단: 실시간 반응 피드 (Reaction Feed)
* 태블릿 클라이언트에서 보낸 이모지 및 한줄평 텍스트 피드를 실시간 갱신 (WebSocket 푸시 수신)
* 스크롤 오버플로우 적용하여 최근 피드가 항상 상단에 위치하도록 갱신

## 3. 디자인 시스템 및 CSS 구현 규칙 (Taste-Skill)

1. **AI 전용 색상 사용 금지 (Lila Ban)**: 보라색 계열(`rgba(139, 92, 246, ...)` 등)의 사용을 엄격히 제한합니다.
2. **단일 액센트(Accent) 지정**: 액센트 컬러는 주황색 계열(#e8620a / 채도 80% 미만) 1개로만 제한하여 일관성을 높입니다.
3. **숫자 모노스페이스**: 테이블 번호, 시간, 대기열 건수 등의 정보에는 수치 정렬이 어긋나지 않도록 `font-variant-numeric: tabular-nums` 혹은 `monospace` 계열의 서체를 사용합니다.
4. **네온 이펙트 및 아웃글로우(Neon glow) 제한**: 버튼이나 카드 테두리에 반짝이는 그림자 효과를 금지하고 깔끔하고 명확한 1px border 디자인을 유지합니다.

## 4. 음원 소스별 재생 처리 워크플로우

1. **Spotify**
   * 재생 동작 시 `spotipy`를 통해 설정된 Spotify Connect 기기로 `play(track_uri)` API 요청
2. **로컬 오디오 (Local File)**
   * 서버 측 static 경로(`/ui-static/images/owned/...`) 및 음원 스트리밍 API를 사용하여 로컬 오디오 렌더링
3. **CD/LP (수동 기기)**
   * 직원이 수동으로 실물 기기를 조작하므로, [재생] 클릭 시 가상의 타이머를 재생 시간만큼 가동하고 태블릿 클라이언트들에게는 "지금 재생 중" WebSocket 메시지를 즉시 전파하여 일치감을 제공함.

## 5. 최적의 작업 실행 스킬 제안 (Optimal Agent Skills)

본 레이아웃 및 제어 스펙을 구현할 때, 단계별 최적의 에이전트 스킬 조합은 다음과 같습니다:

| 구현 단계 | 추천 스킬 (Skill Code) | 추천 사유 |
|---|---|---|
| **Phase 1: DB 모델 보완**<br>(table_device, track_reaction 생성) | `superpowers:executing-plans` | 데이터 마이그레이션 및 SQLite 스키마 변경 시 Contiguous 버전을 지키기 위해 정밀하고 기계적인 순차 코딩이 적합합니다. |
| **Phase 2: WebSocket 서버 강화** | `superpowers:subagent-driven-development` | 연결 관리(Connection Registry), 동시성 제어 및 메시지 전파(Broadcast) 로직은 서브에이전트가 격리된 단위 테스트 환경에서 Mocking 검증을 수행하며 작업하기에 적합합니다. |
| **Phase 3: `/ops/cafe` 프론트엔드 구현** | `design-taste-frontend`<br>(taste-skill 준수) | Taste-Skill CSS 규칙, 비대칭 레이아웃, 모노스페이스 적용, 단일 액센트 배색 등을 정확하게 렌더링하기 위해 UI/UX 특화 스킬이 필수적입니다. |
