기준은 `관리자`와 `현장 운영자`가 같은 도구를 쓰더라도 서로 다른 책임 범위로 움직인다는 점입니다.

## 1. 운영 원칙

핵심 원칙
- 위치 운영은 `대시보드`에서 끝냅니다.
- 메타 수정은 `검색/관리` 또는 `소스 보강`에서 끝냅니다.
- 신규 입력은 `등록/수집`에서 시작합니다.
- 구조 관리, 예외 처리, 백업은 `운영/연계`에서 처리합니다.

역할별 시작점
- 관리자
  `대시보드`, `검색/관리`, `소스 보강`, `등록/수집`, `운영/연계`
- 현장 운영자
  `운영 홈`

기본 작업 순서
1. 상품을 먼저 등록하거나 가져옵니다.
2. 필요하면 외부 소스로 메타를 보강합니다.
3. 내부 마스터를 정리합니다.
4. 마지막에 실제 장식장/칸으로 배치합니다.

## 2. 화면 역할 요약

`대시보드`
- 장식장/층/칸 기준 배치, 이동, 복구

`운영 홈`
- 현장 응대용 검색, 현재 위치/직전 위치 확인, 요청곡 로그

`검색/관리`
- 보유 상품 상세 수정, 커버/트랙/연계 정보 확인, 마스터 연결 확인

`소스 보강`
- 외부 후보 조회 후 기존 상품 메타 교체

`등록/수집`
- 간편 등록, 구매 내역 수입, CSV 대량 입력, 마스터 정리

`운영/연계`
- 시스템 상태, 장식장 구조, 예외 큐, 계정, 백업/복원, 메타 동기화

## 3. 대시보드

목적
- 실제 보관 위치를 운영하는 메인 화면입니다.

언제 쓰는가
- 미배치 상품을 칸으로 넣을 때
- 같은 칸 안에서 순서를 바꿀 때
- 다른 칸 또는 다른 장식장으로 이동할 때
- 잘못 이동한 상품을 직전 위치로 복구할 때

사용 순서
1. 장식장을 선택합니다.
2. 층과 칸을 선택합니다.
3. `선반`, `썸네일`, `리스트` 중 보기 모드를 고릅니다.
4. 상품을 선택합니다.
5. `맨 앞`, `맨 뒤`, 다른 칸 클릭 이동, `직전 위치 복구`를 사용합니다.
6. 우측 이동 작업대에서 `미배치`, `검색 결과`, `최근 이동`을 함께 활용합니다.

주의
- 위치 정리는 여기서 끝내는 것이 가장 안전합니다.
- 선택 카드에서 메타 수정까지 같이 하려고 하지 않는 편이 좋습니다.

## 4. 운영 홈

목적
- 현장 응대용 조회 화면입니다.

언제 쓰는가
- 고객이 특정 곡이나 상품을 찾을 때
- 현재 위치와 직전 위치를 빨리 확인해야 할 때
- 요청곡 기록을 남겨야 할 때

사용 순서
1. 곡명, 아티스트명, 상품명, 바코드, 견출지 ID로 검색합니다.
2. 결과 위 `운영 도우미`에서 정규화 검색어, 상단 후보, 현재 위치를 먼저 확인합니다.
3. 현재 위치가 있으면 `장식장 열기`로 대시보드로 이동합니다.
4. 하단 목록에서 현재 위치와 직전 위치를 다시 확인합니다.
5. 필요하면 요청곡을 등록하고 상태를 업데이트합니다.

운영 도우미 규칙
- 별도 검색 엔드포인트를 추가 호출하지 않습니다.
- 기존 `/operator/catalog-search` 결과를 다시 요약합니다.
- 우선순서는 `바코드 정확 일치 -> 견출지 ID 정확 일치 -> 제목/아티스트 정확 일치 -> 현재 배치된 행 -> 첫 번째 행`입니다.
- 미배치 상품은 `현재 위치 = 미배치`로 표시되고 `장식장 열기`는 숨깁니다.

## 5. 검색/관리

목적
- 보유 상품 상세를 점검하고 수정하는 화면입니다.

언제 쓰는가
- 메타를 직접 고쳐야 할 때
- 같은 마스터의 보유 상품 구성을 봐야 할 때
- 연계 상품, 연계 굿즈, 커버, 곡 정보를 점검할 때

사용 순서
1. 아티스트명, 상품명, 곡명, 카탈로그, 바코드로 검색합니다.
2. 대상 상품을 엽니다.
3. 상품 정보, 커버, 트랙, 연계 정보, 소스 링크 상태를 확인합니다.
4. 필요하면 `정렬 아티스트명`, `권장 보관 규격`, `도메인`, `메모`를 수정합니다.

### 도메인 분류 기준

| 도메인 코드 | 표시명 | 정의 |
|-----------|--------|------|
| `KOREA` | 가요 | 한국 음악 |
| `WESTERN` | 팝/웨스턴 | 영어권 음악 (미국, 영국, 캐나다, 호주 등) |
| `JAPAN` | J-POP | 일본 음악 |
| `GREATER_CHINA` | C-Pop | 중화권 음악 (중국, 대만, 홍콩) |
| `WORLD` | 월드 | 비영어권 서양 + 기타 지역 (프랑스, 스페인, 이탈리아, 브라질, 인도 등) |
| `OTHER_ASIA` | 아시아 | 기타 아시아 (태국, 베트남, 인도네시아 등) |

**판단 기준:**
- 음악/아티스트의 언어권을 기준으로 분류. 발매국(pressing country)은 참고용.
- 영어로 노래하는 독일·스웨덴 밴드는 `WESTERN`.
- 프랑스어·스페인어·포르투갈어·이탈리아어 등 비영어 서양 음악은 `WORLD`.
- 카리브·아프리카·인도 등 비서양 음악도 `WORLD`.

5. 저장 후 마스터 연결 상태를 다시 확인합니다.

주의
- 위치 변경은 여기서 하지 말고 `대시보드`로 이동합니다.
- 같은 앨범의 여러 미디어가 섞여 있으면 먼저 `마스터 정리` 상태를 확인합니다.

## 6. 소스 보강

목적
- 이미 등록된 상품을 Discogs, ManiaDB, Aladin 후보로 보강하는 화면입니다.

언제 쓰는가
- 커버가 없을 때
- 트랙리스트가 비어 있을 때
- 발매일, 레이블, 카탈로그, 바코드 같은 메타가 부족할 때
- 수동 등록 상품을 외부 소스와 연결할 때

기본 흐름
1. 대상 범위를 고릅니다.
   `미연결만`, `전체`, `기연결만`
2. `대상 불러오기`
3. `전체 후보 조회`
4. 자동 선택된 후보를 확인하거나 직접 후보를 고릅니다.
5. `선택 후보 일괄 교체`

주의
- 이 화면은 `기존 상품 보강` 전용입니다.
- 신규 상품 생성은 `등록/수집`에서 처리합니다.

## 7. 등록/수집

### 7-1. 간편 등록

목적
- 최소 정보로 보유 상품을 먼저 만드는 화면입니다.

언제 쓰는가
- 속도가 우선일 때
- 장식장 배치보다 등록 자체를 먼저 끝내야 할 때

사용 순서
1. 포맷, 수량, 아티스트명, 상품명을 입력합니다.
2. 필요하면 발매일, 레이블, 커버 URL, 보관 슬롯을 함께 입력합니다.
3. 저장합니다.
4. 이후 `소스 보강` 또는 `API 메타 조회`로 메타를 채웁니다.

### 7-2. API 메타 조회

목적
- 바코드 또는 무바코드 조회로 외부 메타를 찾아 신규 상품을 저장합니다.

사용 순서
1. 바코드 또는 질의를 입력합니다.
2. 후보를 확인합니다.
3. 적합한 후보를 선택해 저장합니다.
4. 저장 직후 자동 배치하지 말고, 필요하면 나중에 `대시보드`에서 이동합니다.

### 7-3. 구매 내역 가져오기

목적
- 주문 파일이나 메일 본문을 `구매 수입 큐`로 가져오는 화면입니다.

지원 입력
- Amazon 주문 페이지 파일
- eBay 주문 페이지 파일
- Aladin / YES24 / Sailmusic 주문 텍스트 또는 메일
- `.mhtml`, `.mht`, `.html`, `.htm`, `.txt`

기본 흐름
1. 파일을 선택하거나 본문을 준비합니다.
2. `미리보기`로 파싱 결과를 확인합니다.
3. `수입 큐 저장`으로 큐에 적재합니다.
4. `구매 수입 큐`에서 `후보 조회` 또는 `후보 일괄 조회`를 실행합니다.
5. 적합한 후보를 선택해 `후보로 생성` 또는 직접 생성합니다.
6. 생성된 상품은 기본적으로 `미배치`로 두고, 이후 대시보드에서 위치를 정합니다.

확인할 것
- 커버
- 아티스트명
- 상품명
- 미디어 포맷
- 수량
- 가격
- 후보 수

더 자세한 설정은 [purchase_mail_import.md](/Volumes/Data/Works/07.__PROJECT_SLUG__/docs/purchase_mail_import.md)를 봅니다.

### 7-4. 대량 등록

목적
- CSV 기반 일괄 입력입니다.

언제 쓰는가
- 기존 소장품을 한 번에 넣을 때
- Discogs release id, 바코드, 위치 정보가 미리 정리돼 있을 때

사용 순서
1. CSV 파일을 선택합니다.
2. 필요하면 기본 카테고리와 메모를 입력합니다.
3. 업로드합니다.
4. 자동 확정되지 않은 항목은 `검수 큐`에서 처리합니다.

주의
- CSV 업로드는 자동 등록이 아니라 `검수 큐` 중심 흐름입니다.
- 위치 컬럼과 release id가 정확할수록 수작업이 줄어듭니다.
- 예제 파일은 [csv_import_sample.csv](/Volumes/Data/Works/07.__PROJECT_SLUG__/docs/csv_import_sample.csv)를 기준으로 맞춥니다.

### 7-5. 마스터 정리

목적
- 같은 앨범의 여러 보유 상품을 하나의 내부 마스터로 묶는 화면입니다.

언제 쓰는가
- LP, CD, 재발매, 수입반이 따로 흩어져 있을 때
- 예외 큐의 `마스터 없음`을 정리할 때

사용 순서
1. 외부 마스터를 검색합니다.
2. 마스터를 선택합니다.
3. 필요한 경우 해당 마스터의 버전을 확인합니다.
4. 보유 상품을 선택합니다.
5. 마스터에 묶습니다.

주의
- 같은 앨범을 서로 다른 내부 마스터에 나눠 두지 않는 것이 중요합니다.
- `정렬 아티스트명`과 도메인 보정은 마스터 기준으로 보는 것이 좋습니다.

## 8. 운영/연계

### 8-1. 시스템 상태

목적
- 앱 상태, 외부 접속, launchd 로그, QA 요약을 한 곳에서 확인합니다.

문제 발생 시 순서
1. `/health`가 `200`인지 확인합니다.
2. launchd 로그 최근 2줄을 확인합니다.
3. 필요하면 `./scripts/check_library_status.sh --short`를 실행합니다.

### 8-2. 장식장 / 슬롯

목적
- 장식장 구조와 예외 슬롯을 관리합니다.

원칙
- 기본 구조는 장식장 등록에서 일괄 생성합니다.
- 개별 슬롯 수정은 비정형 구조나 예외 칸에만 씁니다.
- 정렬 정책은 장식장 단위입니다.
  `ARTIST_RELEASE_TITLE`, `LABEL_ID`

운영 팁
- 전시 장식장은 `ARTIST_RELEASE_TITLE`
- 후면 보관/증설 장식장은 `LABEL_ID`
로 나누면 재배치량이 줄어듭니다.

### 8-3. 예외 큐

목적
- 병목 상품만 모아 빠르게 처리합니다.

우선순위
1. 미배치
2. 소스 미연결
3. 마스터 없음
4. 커버 없음
5. 보관 규격 예외
6. 곡 정보 없음

사용 순서
1. 예외 종류를 선택합니다.
2. 표시 수를 정합니다.
3. 필요하면 저장된 필터를 적용합니다.
4. 다중 선택 후 `소스 보강`, `실물 기준 맞춤`, `마스터 정리`로 보냅니다.

### 8-4. 계정

목적
- 관리자와 현장 운영자 계정을 관리합니다.

원칙
- 모든 계정은 `auth_account` 테이블에서 관리합니다.
- `.env.local`의 `LIBRARY_ADMIN_*`, `LIBRARY_OPERATOR_*`, `LIBRARY_OPERATOR_ACCOUNTS`는 **부트스트랩 1회 시드용**입니다. 첫 부팅 시 자동으로 DB에 해시 저장되며, 이후 로그인 검증은 DB만 참조합니다.
- 비밀번호 변경은 이 화면(또는 `/admin/auth-accounts` API)에서만 진행합니다. ENV 값을 바꿔도 이미 시드된 계정의 비밀번호는 변경되지 않습니다.
- 시드가 끝난 뒤에는 `.env.local`에서 평문 자격증명을 제거(또는 회전)하고, 운영자는 이 화면에서만 새 계정을 생성/수정합니다.
- 현장 운영자는 조회/요청 중심, 관리자는 수정 가능 범위로 운영합니다.

### 8-5. 백업 / 내보내기

목적
- 대량 작업 전 보호 장치와 운영용 산출물 추출입니다.

권장 절차
1. 대량 수정 전 `DB 백업 다운로드`
2. 운영 공유용 `보유 상품 CSV`, `마스터 CSV`
3. 필요하면 QA 시트나 수입 큐 증적과 함께 보관

### 8-6. 메타 동기화

목적
- 이미 연결된 외부 소스 상품의 누락 메타를 후속 보강합니다.

원칙
- 기본은 `누락 필드만 보강`
- 전체 동기화는 필요할 때만 실행

## 9. 자주 쓰는 실제 운영 시나리오

신규 입고
1. `등록/수집`에서 간편 등록 또는 구매 내역 수입으로 상품을 생성합니다.
2. `소스 보강`으로 커버/트랙/발매 정보를 보강합니다.
3. `마스터 정리`로 내부 마스터를 맞춥니다.
4. `대시보드`에서 장식장에 배치합니다.

고객이 특정 곡을 찾는 경우
1. `운영 홈`에서 검색합니다.
2. `운영 도우미`의 상단 후보와 현재 위치를 먼저 봅니다.
3. 필요하면 `장식장 열기`로 대시보드로 이동합니다.
4. 응대 후 요청곡 상태를 정리합니다.

미배치/커버 없음 정리
1. `운영/연계 > 예외 큐`에서 병목 종류를 고릅니다.
2. 일괄로 `소스 보강` 또는 `마스터 정리`로 보냅니다.
3. 마지막에 `대시보드`에서 실제 배치를 끝냅니다.

## 10. 운영 한 줄 정리

- 위치는 `대시보드`
- 현장 응대는 `운영 홈`
- 메타 수정은 `검색/관리`
- 외부 메타 보강은 `소스 보강`
- 신규 입력은 `등록/수집`
- 구조, 예외, 계정, 백업은 `운영/연계`
