# 라이브러리 관리툴 운영 매뉴얼

이 문서는 현재 운영 콘솔의 화면 역할과 실제 작업 순서를 정리한 최종 운영 매뉴얼입니다.  
기준은 `관리자`와 `현장 운영자`가 같은 도구를 쓰더라도 서로 다른 책임 범위로 움직인다는 점입니다.

기준 주소
- 운영 로그인: `https://__PROD_DOMAIN__/login`
- 헬스체크: `https://__PROD_DOMAIN__/health`

관련 문서
- 운영자용 ERD 요약: [library_erd_operator.md](library_erd_operator.md)
- 개발자용 ERD 상세: [library_erd.md](library_erd.md)
- 구매 내역 수입 가이드: [purchase_mail_import.md](purchase_mail_import.md)
- 상용화 체크리스트: [go_live_checklist.md](go_live_checklist.md)
- macOS QA/운영 런북: [macos_qa_prod_runbook.md](macos_qa_prod_runbook.md)

## 1. 운영 원칙

핵심 원칙
- 위치 운영은 `대시보드`에서 끝냅니다.
- 메타 수정은 `검색/관리` 또는 `소스 보강`에서 끝냅니다.
- 신규 입력은 `등록/수집`에서 시작합니다.
- 구조 관리, 예외 처리, 백업은 `운영/연계`에서 처리합니다.
- 카페 손님 응대는 `카페`에서 처리합니다.

역할별 시작점
- 관리자: `대시보드`, `검색/관리`, `소스 보강`, `등록/수집`, `굿즈 관리`, `운영/연계`, `카페 관리`
- 현장 운영자: `운영 홈`, `카페`

기본 작업 순서
1. 상품을 먼저 등록하거나 가져옵니다.
2. 필요하면 외부 소스로 메타를 보강합니다.
3. 내부 마스터를 정리합니다.
4. 마지막에 실제 장식장/칸으로 배치합니다.

## 2. 화면 역할 요약

`대시보드`
- 장식장/층/칸 기준 배치, 이동, 복구, 컬렉션 통계, 랜덤 앨범

`운영 홈`
- 현장 응대용 검색, 현재 위치/직전 위치 확인, 요청곡 로그, Roon 재생 상태, 사무실 온습도, 아티스트 컨텍스트

`검색/관리`
- 보유 상품 상세 수정, 커버/트랙/연계 정보 확인, 마스터 연결, 장르/스타일, 앨범 리뷰, Spotify 연동, 이미지 업로드, 디지털 링크

`소스 보강`
- 외부 후보 조회 후 기존 상품 메타 교체, Aladin 이미지 백필, 이미지 새로고침, Discogs 콜렉터 정보

`등록/수집`
- 간편 등록, 바코드 스캔, 검색 등록, 구매 내역 수입, CSV, 마스터 정리, 검수 큐

`굿즈 관리`
- 굿즈 상품 검색, 등록, 수정, 삭제, 상품 연계 매핑

`운영/연계`
- 시스템 상태, 장식장 구조, 예외 큐, 계정, 백업/복원, 메타 동기화, 프로바이더 설정, 배치 힌트, 상품 그룹, 감사 로그, 장식장 카메라

`카페`
- 고객 요청곡 접수/큐, 현재 재생 중, 플레이리스트, 가사 보기, 리액션, Spotify/로컬 재생

`카페 관리`
- 재생 디바이스 등록/관리, 카페 통계

## 3. 대시보드

목적
- 실제 보관 위치를 운영하는 메인 화면입니다.

언제 쓰는가
- 미배치 상품을 칸으로 넣을 때
- 같은 칸 안에서 순서를 바꿀 때
- 다른 칸 또는 다른 장식장으로 이동할 때
- 잘못 이동한 상품을 직전 위치로 복구할 때
- 컬렉션 통계를 한눈에 볼 때

사용 순서
1. 장식장을 선택합니다.
2. 층과 칸을 선택합니다.
3. `선반`, `썸네일`, `리스트` 중 보기 모드를 고릅니다.
4. 상품을 선택합니다.
5. `맨 앞`, `맨 뒤`, 다른 칸 클릭 이동, `직전 위치 복구`를 사용합니다.
6. 우측 이동 작업대에서 `미배치`, `검색 결과`, `최근 이동`을 함께 활용합니다.

컬렉션 대시보드
- 장르/도메인별 보유 통계를 확인할 수 있습니다.
- `랜덤 앨범`으로 무작위 음반을 뽑을 수 있습니다.

주의
- 위치 정리는 여기서 끝내는 것이 가장 안전합니다.
- 선택 카드에서 메타 수정까지 같이 하려고 하지 않는 편이 좋습니다.

## 4. 운영 홈

목적
- 현장 응대용 조회 화면입니다.

언제 쓰는가
- 고객이 특정 곡이나 상품을 찾을 때
- 현재 위치와 직전 위치를 빨리 확인해야 할 때
- 요청곡 기록을 남겨야 할 때
- Roon 연동 상태를 확인할 때
- 사무실 온습도를 모니터링할 때

사용 순서
1. 곡명, 아티스트명, 상품명, 바코드, 견출지 ID로 검색합니다.
2. 결과 위 `운영 도우미`에서 정규화 검색어, 상단 후보, 현재 위치를 먼저 확인합니다.
3. 현재 위치가 있으면 `장식장 열기`로 대시보드로 이동합니다.
4. 하단 목록에서 현재 위치와 직전 위치를 다시 확인합니다.
5. 필요하면 요청곡을 등록하고 상태를 업데이트합니다.

운영 도우미 규칙
- 별도 검색 엔드포인트를 추가 호출하지 않습니다.
- 기존 검색 결과를 다시 요약합니다.
- 우선순서는 `바코드 정확 일치 → 견출지 ID 정확 일치 → 제목/아티스트 정확 일치 → 현재 배치된 행 → 첫 번째 행`입니다.
- 미배치 상품은 `현재 위치 = 미배치`로 표시되고 `장식장 열기`는 숨깁니다.

Roon 연동
- Roon Server의 현재 재생 상태를 확인할 수 있습니다.
- Roon에서 재생 중인 곡을 요청곡과 연결할 수 있습니다.

사무실 환경
- 사무실 온습도를 실시간으로 확인할 수 있습니다.
- 외부 기온과 비교한 쾌적도 정보를 제공합니다.

아티스트 컨텍스트
- 아티스트명으로 AI 기반 배경 정보(장르, 시대, 유사 아티스트 등)를 조회할 수 있습니다.

## 5. 검색/관리

목적
- 보유 상품 상세를 점검하고 수정하는 화면입니다.

언제 쓰는가
- 메타를 직접 고쳐야 할 때
- 같은 마스터의 보유 상품 구성을 봐야 할 때
- 연계 상품, 연계 굿즈, 커버, 곡 정보를 점검할 때
- 장르/스타일을 관리할 때
- 앨범 리뷰를 작성할 때

사용 순서
1. 아티스트명, 상품명, 곡명, 카탈로그, 바코드로 검색합니다.
2. 대상 상품을 엽니다.
3. 상품 정보, 커버, 트랙, 연계 정보, 소스 링크 상태를 확인합니다.
4. 필요하면 `정렬 아티스트명`, `권장 보관 규격`, `도메인`, `메모`, `장르/스타일`을 수정합니다.
5. 저장 후 마스터 연결 상태를 다시 확인합니다.

### 도메인 분류 기준

| 도메인 코드 | 표시명 | 정의 |
|-----------|--------|------|
| `KOREA` | 가요 | 한국 음악 |
| `WESTERN` | 팝/웨스턴 | 영어권 음악 |
| `JAPAN` | J-POP | 일본 음악 |
| `GREATER_CHINA` | C-Pop | 중화권 음악 |
| `WORLD` | 월드 | 비영어권 서양 + 기타 지역 |
| `OTHER_ASIA` | 아시아 | 기타 아시아 |

판단 기준: 음악/아티스트의 언어권 기준. 영어로 노래하는 유럽 밴드는 `WESTERN`, 비영어 서양 음악은 `WORLD`.

### 장르/스타일
- 마스터 편집 화면에서 장르와 스타일 태그를 추가/제거할 수 있습니다.
- 메타 동기화 시 Discogs 등 외부 소스의 장르 정보가 자동 반영됩니다.

### 앨범 리뷰
- `자동 생성`: AI가 앨범 정보를 바탕으로 한국어 리뷰를 생성합니다.
- `URL 기반`: 외부 리뷰 URL을 첨부할 수 있습니다.
- `직접 작성`: 수동으로 리뷰를 작성할 수 있습니다.
- `일괄 미리보기`: 여러 앨범을 선택해 한 번에 생성 결과를 미리보고 확정할 수 있습니다.

### Spotify 연동
- 앨범 마스터를 Spotify 앨범과 매칭할 수 있습니다.
- 매칭된 앨범은 Spotify에서 바로 재생할 수 있습니다.
- 매칭 해제: 마스터 패널 Spotify 편집 모드에서 `삭제` 버튼 클릭 → `album_master.spotify_album_id` 초기화

### 이미지 관리
- 상품 이미지를 직접 업로드하거나 URL로 추가할 수 있습니다.
- 여러 장의 이미지를 등록하고 순서를 관리할 수 있습니다.

### 디지털 링크
- 로컬 음악 파일 경로나 스트리밍 링크를 상품에 연결할 수 있습니다.

주의
- 위치 변경은 여기서 하지 말고 `대시보드`로 이동합니다.
- 같은 앨범의 여러 미디어가 섞여 있으면 먼저 `마스터 정리` 상태를 확인합니다.

## 6. 소스 보강

목적
- 이미 등록된 상품을 Discogs, ManiaDB, Aladin 후보로 보강하는 화면입니다.

언제 쓰는가
- 커버가 없을 때, 트랙리스트가 비어 있을 때
- 발매일, 레이블, 카탈로그, 바코드 같은 메타가 부족할 때
- 수동 등록 상품을 외부 소스와 연결할 때

기본 흐름
1. 대상 범위 선택: `미연결만`, `전체`, `기연결만`
2. `대상 불러오기`
3. `전체 후보 조회`
4. 자동 선택된 후보 확인 또는 직접 후보 선택
5. `선택 후보 일괄 교체`

Aladin 이미지 백필
- Aladin API를 통해 커버 이미지를 일괄 보강할 수 있습니다.
- 진행 상태를 확인하고 필요하면 중단할 수 있습니다.

이미지 새로고침
- 이미 연결된 소스의 커버 이미지를 최신 상태로 갱신합니다.

Discogs 콜렉터 정보
- Discogs release의 커뮤니티 통계(보유 수, 평가 등)를 조회할 수 있습니다.

주의
- 이 화면은 기존 상품 보강 전용입니다. 신규 상품 생성은 `등록/수집`에서 처리합니다.

## 7. 등록/수집

### 7-1. 간편 등록
목적: 최소 정보로 보유 상품을 먼저 만듭니다.
순서: 포맷, 수량, 아티스트명, 상품명 입력 → 저장 → 이후 소스 보강으로 메타 채움

### 7-2. 바코드 스캔 등록
목적: 바코드 스캔으로 외부 메타를 찾아 신규 상품을 저장합니다.
순서: 바코드 스캔 → 후보 확인 → 선택 저장 → 필요시 배치 위치 추천 참고

### 7-3. 검색 등록
목적: 아티스트명/앨범명으로 외부 소스 검색 후 등록합니다.
순서: 검색어 입력 + 소스 선택(Discogs, ManiaDB, Aladin) → 후보 선택 → 추가 정보 입력 후 저장

### 7-4. 구매 내역 가져오기
목적: 주문 파일이나 메일 본문을 구매 수입 큐로 가져옵니다.
지원: Amazon, eBay, Aladin, YES24, Sailmusic (.mhtml, .mht, .html, .htm, .txt)
순서: 파일 선택 → 미리보기 → 수입 큐 저장 → 후보 조회 → 후보로 생성 → 대시보드에서 배치
상세: [purchase_mail_import.md](purchase_mail_import.md)

### 7-5. 대량 등록 (CSV)
목적: CSV 기반 일괄 입력입니다.
순서: CSV 파일 선택 → 업로드 → 검수 큐에서 확정
예제: [csv_import_sample.csv](csv_import_sample.csv)

### 7-6. 검수 큐
목적: CSV 업로드나 자동 등록 후 확인이 필요한 항목을 처리합니다.
순서: 검수 대상 확인 → 메타 검토/수정 → 확정

### 7-7. 마스터 정리
목적: 같은 앨범의 여러 보유 상품을 하나의 내부 마스터로 묶습니다.
순서: 외부 마스터 검색 → 마스터 선택 → 버전 확인 → 보유 상품 선택 → 마스터에 묶기
주의: 같은 앨범을 서로 다른 내부 마스터에 나눠 두지 않습니다.

## 8. 굿즈 관리

목적
- 음반 외 굿즈 상품을 별도 관리하는 화면입니다.

언제 쓰는가
- 포스터, 포토카드, 응원봉 등 비음반 상품을 등록/관리할 때
- 굿즈를 음반 마스터와 연계할 때

사용 순서
1. 굿즈 검색 또는 신규 등록
2. 상품 정보(이름, 카테고리, 수량, 위치 등) 입력
3. 필요시 연계할 음반 마스터 매핑
4. 상품 간 관계 설정 (세트 구성 등)

## 9. 카페

목적
- 오프라인 음악 카페 운영을 위한 고객 응대 화면입니다.

### 9-1. 요청곡 접수
- 고객이 듣고 싶은 곡을 검색해 요청을 접수합니다.
- 요청곡은 큐에 쌓이고 운영자가 순서대로 처리합니다.
- 요청 상태: 대기 → 재생 중 → 완료 / 취소

### 9-2. 현재 재생 중
- 현재 재생 중인 곡 정보를 실시간으로 표시합니다.
- WebSocket으로 연결되어 별도 새로고침 없이 갱신됩니다.

### 9-3. 플레이리스트
- 운영자가 미리 구성한 재생 목록을 관리합니다.
- 플레이리스트 생성, 곡 추가/삭제, 즉시 재생이 가능합니다.

### 9-4. 가사 보기
- 현재 재생 중인 곡의 가사를 조회할 수 있습니다.

### 9-5. 리액션
- 고객이 현재 재생 중인 곡에 좋아요 등의 반응을 남길 수 있습니다.

### 9-6. 재생 제어
- Spotify 연동: Spotify 프리미엄 계정으로 트랙 재생
- 로컬 재생: 로컬 음악 파일 직접 재생/일시정지/정지
- 재생 디바이스 지정 가능

## 11. 운영/연계

### 11-1. 시스템 상태
목적: 앱 상태, 외부 접속, launchd 로그, QA 요약을 한 곳에서 확인합니다.
문제 발생 시: /health 확인 → launchd 로그 확인 → check_library_status.sh 실행

### 11-2. 장식장 / 슬롯
목적: 장식장 구조와 예외 슬롯을 관리합니다.
원칙: 기본 구조는 장식장 등록에서 일괄 생성. 개별 슬롯 수정은 비정형 구조나 예외 칸에만 사용.
정렬 정책: `ARTIST_RELEASE_TITLE`(전시용), `LABEL_ID`(보관용)

### 11-3. 장식장 카메라
목적: ONVIF 호환 IP 카메라를 장식장에 연결해 실시간 모니터링합니다.
순서: 카메라 검색(ONVIF Discover) → 연결 테스트 → 등록 → 스냅샷 확인

### 11-4. 예외 큐
목적: 병목 상품만 모아 빠르게 처리합니다.
순서: 예외 종류 선택 → 표시 수 지정 → 다중 선택 → 소스 보강/마스터 정리로 이동

예외 13종 (★ 마스터 기준 — 카드 클릭 시 마스터 패널 열림)

| 예외 종류 | 처리 방법 |
|-----------|----------|
| `UNSLOTTED` | 대시보드에서 장식장 배치 |
| `SOURCE_MISSING` | 소스 코드·ID 직접 입력 |
| `MASTER_MISSING` | 마스터 정리에서 연결 |
| `COVER_MISSING` | 소스 보강 또는 이미지 추가 |
| `PREFERRED_SIZE_MISMATCH` | 권장 보관 규격 수정 |
| `MEDIA_MISSING` | 미디어 형식 입력 |
| `SIZE_MISMATCH` | 슬롯 이동 또는 사이즈 수정 |
| `TRACK_MISSING` | 트랙 정보 추가 |
| ★ `SPOTIFY_UNMATCHED` | 마스터 패널에서 Spotify 매칭 |
| ★ `REVIEW_MISSING` | 마스터 패널에서 리뷰 작성/URL 입력 |
| ★ `GENRE_MISSING` | 마스터 패널에서 장르 추가 |
| `CATALOG_MISSING` | 카탈로그 번호 직접 입력 |
| ★ `LOCAL_MISSING` | 마스터 패널에서 NAS 디렉터리 연결 |

마스터 기준 예외는 마스터 1개를 수정하면 해당 마스터에 연결된 모든 보유 상품의 예외가 한 번에 해소됩니다.

### 11-5. 계정
목적: 관리자와 현장 운영자 계정을 관리합니다.
원칙:
- 모든 계정은 `auth_account` 테이블에서 관리합니다.
- 최초 부트스트랩 시 .env의 시드 계정이 DB에 해시 저장됩니다.
- 이후 로그인 검증은 DB만 참조하며, 계정 관리 화면에서만 비밀번호를 변경합니다.

### 11-6. 백업 / 내보내기
목적: 대량 작업 전 보호 장치와 운영용 산출물 추출입니다.
기능: DB 백업 다운로드, 전체 백업, 보유 상품 CSV, 마스터 CSV, 백업 설정(주기, 대상)

### 11-7. 메타 동기화
목적: 이미 연결된 외부 소스 상품의 누락 메타를 후속 보강합니다.
원칙: 기본은 누락 필드만 보강. 전체 동기화는 필요할 때만 실행.

### 11-8. 프로바이더 설정
목적: Discogs, Aladin, DeepL 등 외부 API 키와 설정을 관리합니다.
기능: 토큰 저장, 연결 테스트(DeepL)

### 11-9. 배치 힌트
목적: 신규 입고 상품의 적절한 배치 위치를 AI가 추천합니다.

### 11-10. 상품 그룹
목적: 테마별 상품 그룹(예: "비 오는 날", "겨울 재즈")을 만들어 관리합니다.

### 11-11. 감사 로그
목적: 주요 작업(메타 수정, 위치 변경, 계정 변경 등)의 이력을 추적합니다.

## 12. 자주 쓰는 실제 운영 시나리오

신규 입고
1. `등록/수집`에서 간편 등록 또는 구매 내역 수입으로 상품 생성
2. `소스 보강`으로 커버/트랙/발매 정보 보강
3. `마스터 정리`로 내부 마스터 맞춤
4. `대시보드`에서 장식장에 배치

고객이 특정 곡을 찾는 경우
1. `운영 홈`에서 검색
2. `운영 도우미`의 상단 후보와 현재 위치 확인
3. 필요하면 `장식장 열기`로 대시보드 이동
4. 응대 후 요청곡 상태 정리

카페에서 요청곡 접수
1. `카페` 화면에서 고객 검색
2. 요청곡 접수 → 큐 등록
3. `카페 관리` 디바이스로 재생
4. 재생 완료 후 상태 업데이트

미배치/커버 없음 정리
1. `운영/연계 > 예외 큐`에서 병목 종류 선택
2. 일괄로 `소스 보강` 또는 `마스터 정리`로 이동
3. 마지막에 `대시보드`에서 실제 배치

로컬 플레이어 연결 (LOCAL_MISSING 해소)
1. `운영/연계 > 예외 큐`에서 `LOCAL_MISSING` 선택
2. 카드 클릭 → 마스터 패널에서 NAS 디렉터리 탐색
3. 연결 후 `운영 홈`에서 트랙 목록 확인 및 재생

## 13. 운영 한 줄 정리

- 위치는 `대시보드`
- 현장 응대는 `운영 홈`
- 메타 수정은 `검색/관리`
- 외부 메타 보강은 `소스 보강`
- 신규 입력은 `등록/수집`
- 굿즈는 `굿즈 관리`
- 카페 응대는 `카페`
- 구조, 예외, 계정, 백업은 `운영/연계`
